Introduction

Git is a powerful distributed version control system that helps developers track changes in their code. GitHub is a cloud-based platform that leverages Git for collaboration, enabling teams to work on projects together effectively.

  • Why use Git? Version control, collaboration, and change tracking.
  • Why GitHub? Centralized repositories, issue tracking, pull requests, and more.

Setting Up Git

To start using Git, you need to install it and configure it for your environment.

  1. Download and install Git from Git's official website.
  2. Verify the installation:
    3. git --version
  3. Configure your username and email (used in commits):
    4. git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
  4. Set up your default editor (optional):
    5. git config --global core.editor "code --wait"

Using GitHub

GitHub extends Git with collaborative tools:

Creating a Repository

  1. Log in to GitHub.
  2. Click on "New Repository".
  3. Enter a name and description, then create the repository.

Cloning a Repository

git clone https://github.com/username/repo-name.git

Submitting Pull Requests

  1. Fork a repository you want to contribute to.
  2. Clone your forked repository.
  3. Make changes and push them to your fork.
  4. Submit a pull request on GitHub.

Advanced Git Commands

Here are some advanced Git features and commands:

Branching

  • Create a new branch:
    • git branch branch-name
  • Switch to a branch:
    • git checkout branch-name
  • Create and switch to a branch in one step:
    • git checkout -b branch-name

Stashing

Temporarily save changes without committing:

git stash

Apply stashed changes:

git stash apply

Rebasing

Rebase your branch onto another branch:

git rebase branch-name

Tips & Best Practices

  • Commit often with clear, descriptive messages.
  • Use branches for feature development to avoid conflicts.
  • Regularly pull the latest changes from the main branch.
  • Review your changes before committing with git diff.
  • Keep your repository organized and clean by deleting old branches.
